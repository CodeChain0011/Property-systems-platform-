# BLUEPRINT 006: INSPECTION TO WORK ORDER

## Purpose

Map how inspection findings become actionable repairs.

## Flow

1. Inspection is scheduled.
2. Checklist is generated.
3. Inspector completes inspection.
4. Findings are recorded.
5. Photos/videos are uploaded.
6. System flags failed items.
7. Manager reviews findings.
8. Work orders are generated.
9. Vendor is assigned.
10. Corrections are completed.
11. Inspection is closed.

## Records Created

- Inspection
- Finding
- Document
- Work Order
- Communication
- Event
- Audit Log

## Automation Opportunities

- Auto-create work order from failed item
- Asset risk scoring
- Compliance deadline alerts
