# BLUEPRINT 002: MAINTENANCE REQUEST TO CLOSE

## Purpose

Map a maintenance request from creation to completion.

## Flow

1. Tenant reports issue.
2. System creates work order.
3. Issue is categorized.
4. Priority is assigned.
5. Manager reviews.
6. Vendor is assigned.
7. Tenant is notified.
8. Vendor schedules visit.
9. Work is completed.
10. Photos and invoice are uploaded.
11. Manager approves completion.
12. Cost is recorded.
13. Work order closes.
14. Asset history updates.

## Records Created

- Work Order
- Communication
- Document
- Invoice
- Financial Transaction
- Asset History
- Event
- Audit Log

## Automation Opportunities

- Emergency routing
- Vendor assignment suggestions
- Overdue repair alerts
- Cost anomaly detection
