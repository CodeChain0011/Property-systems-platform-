# BLUEPRINT 007: VENDOR INVOICE PAYMENT

## Purpose

Map vendor completion, invoice review, and payment.

## Flow

1. Vendor completes work.
2. Vendor uploads invoice.
3. Vendor uploads completion evidence.
4. Manager reviews invoice.
5. System compares estimate vs actual.
6. Manager approves or rejects invoice.
7. Payment transaction is created.
8. Vendor payment is recorded.
9. Work order financials update.
10. Vendor performance history updates.

## Records Created

- Invoice
- Document
- Financial Transaction
- Vendor History
- Event
- Audit Log

## Automation Opportunities

- Invoice mismatch alerts
- Vendor performance scoring
- 1099 tracking
