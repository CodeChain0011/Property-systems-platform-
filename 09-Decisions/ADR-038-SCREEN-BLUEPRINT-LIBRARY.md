# ADR-038 SCREEN BLUEPRINT LIBRARY

## Decision

The platform will use a screen blueprint library to connect workflows, data models, permissions, and user experience.

## Reason

The Property Operating System contains many roles and workflows. Screens must be intentionally designed around business operations, not improvised during development.

## Consequence

Each major interface should define purpose, data sources, actions, permissions, and mobile behavior before development.

## Status

Accepted
