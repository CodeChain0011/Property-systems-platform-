# ADR-049 GO NO-GO CODING CHECKLIST

## Decision

The project will use a formal go/no-go checklist before beginning MVP coding.

## Reason

The Property Operating System is broad and could easily enter development too early, causing rework, scope creep, and architectural drift.

## Consequence

Coding begins only after product scope, data model, permissions, workflows, screens, APIs, testing, deployment, and developer handoff are sufficiently defined.

## Status

Accepted
