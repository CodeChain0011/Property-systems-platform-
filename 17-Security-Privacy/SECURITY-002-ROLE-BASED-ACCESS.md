# SECURITY 002: ROLE-BASED ACCESS CONTROL

## Purpose

Defines role-based access control.

## Core Roles

- Owner
- Admin
- Property Manager
- Tenant
- Applicant
- Vendor
- Inspector
- Accountant
- Attorney
- Maintenance Worker

## Access Rule

A user should only access the minimum data needed to perform their role.

## Examples

- Tenant can view their lease, payments, documents, messages, and maintenance requests.
- Vendor can view assigned work orders only.
- Accountant can view financial records but not private tenant communications unless authorized.
- Attorney can view legal documents and notices when assigned.
- Owner can view portfolio-level records.
