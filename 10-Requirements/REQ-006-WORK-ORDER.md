# REQ 006: WORK ORDER

## Requirement ID

REQ-WO-001

## User Role

Owner, Property Manager, or Tenant

## Goal

Create and track a maintenance request.

## Main Flow

1. User opens Maintenance.
2. User selects New Work Order.
3. User selects property/unit.
4. User enters issue details.
5. User sets priority.
6. System creates Work Order.
7. System links Work Order to Unit and Property.
8. System creates Event.
9. System creates Audit Log.

## Data Created

- Work Order
- Event
- Audit Log

## Acceptance Criteria

- Work order can be created.
- Work order appears on maintenance board.
- Status can be updated.
- Audit log is created.

## Priority

High
