# WORKFLOW AUTOMATION 001

## Purpose

This document defines how automation should be designed across platform workflows.

## Automation Types

1. Reminder automation
2. Status-change automation
3. Document generation automation
4. Notification automation
5. Payment automation
6. Compliance deadline automation
7. Work order routing automation
8. Renewal recommendation automation
9. Inspection scheduling automation
10. Reporting automation

## Automation Rule

Automation should reduce repetitive work but must not remove required human approval from legal, financial, or high-risk decisions.

## Human Approval Required For

- Lease approval
- Applicant denial
- Eviction notice
- Large vendor payment
- Security deposit deduction
- Rent increase
- Legal notice
- Permission change
