# WORKFLOW REQUIREMENTS 001

## Purpose

This document defines the standard structure every workflow must follow in the Property Operating System.

## Required Workflow Sections

Every workflow document must include:

1. Purpose
2. Trigger
3. Main actors
4. Preconditions
5. Workflow steps
6. Records created
7. Records updated
8. Notifications sent
9. Documents generated
10. Financial impact
11. Compliance impact
12. Audit log requirements
13. Failure states
14. Edge cases
15. AI assistance opportunities

## Workflow Design Rule

No workflow should exist only as a screen.

Every workflow must create or update system records.
