# WORKFLOW 001: APPLICANT TO TENANT

## Purpose

Defines how a prospective applicant becomes an approved tenant with an active lease.

## Trigger

A person submits a rental application.

## Main Actors

- Applicant
- Property Manager
- Owner
- Screening Provider
- System

## Workflow Steps

1. Applicant selects available unit.
2. Applicant submits application.
3. System creates Application record.
4. System links Applicant to Unit and Property.
5. Screening process begins.
6. Applicant uploads required documents.
7. Manager reviews application.
8. Decision record is created.
9. If approved, Lease draft is generated.
10. Applicant signs lease.
11. Owner or manager countersigns.
12. Security deposit is collected.
13. Lease status becomes Active.
14. Applicant role changes to Tenant.
15. Unit status changes to Occupied or Pending Move-In.

## Records Created

- Person
- Role
- Application
- Screening Record
- Document
- Lease
- Financial Transaction
- Communication
- Event
- Audit Log

## Compliance Notes

Application decisions must be auditable and fair housing aware.
