# WORKFLOW 004: MOVE-OUT

## Purpose

Defines how a tenant leaves a unit and how the unit returns to available status.

## Trigger

Lease ends, tenant gives notice, or tenancy is terminated.

## Main Actors

- Tenant
- Property Manager
- Owner
- Inspector
- Vendor
- System

## Workflow Steps

1. Move-out date is confirmed.
2. System creates move-out checklist.
3. Tenant receives move-out instructions.
4. Final inspection is scheduled.
5. Tenant returns keys or access credentials.
6. Move-out inspection is completed.
7. Photos and videos are compared to move-in condition.
8. Damages are documented.
9. Deposit deductions are calculated.
10. Security deposit reconciliation is prepared.
11. Work Orders are created for turnover repairs.
12. Unit status becomes Turnover or Available.

## Records Created

- Inspection
- Document
- Work Order
- Financial Transaction
- Communication
- Event
- Audit Log
