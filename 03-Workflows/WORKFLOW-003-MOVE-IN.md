# WORKFLOW 003: MOVE-IN

## Purpose

Defines how a tenant officially takes possession of a unit.

## Trigger

Lease is signed and move-in date is scheduled.

## Main Actors

- Tenant
- Property Manager
- Owner
- Inspector
- System

## Workflow Steps

1. Lease status becomes signed.
2. Move-in date is confirmed.
3. System creates move-in checklist.
4. Initial inspection is scheduled.
5. Tenant receives access instructions.
6. Keys, codes, or smart lock access are issued.
7. Move-in inspection is completed.
8. Photos and videos are uploaded.
9. Condition record is stored.
10. Tenant confirms possession.
11. Unit status becomes Occupied.

## Records Created

- Inspection
- Document
- Communication
- Event
- Audit Log
- Unit Status Update
