# PROPERTY OPERATING SYSTEM

## FOUNDATION DRAFT 004

### Workflow Architecture

This document defines the core operational workflows that the platform must eventually support.

## Primary Workflows

1. Property onboarding
2. Unit setup
3. Applicant intake
4. Tenant screening
5. Lease generation
6. Move-in inspection
7. Rent collection
8. Maintenance request
9. Work order dispatch
10. Vendor payment
11. Preventive maintenance
12. Lease renewal
13. Move-out inspection
14. Security deposit reconciliation
15. Legal notice generation
16. Eviction tracking
17. Owner reporting
18. Portfolio performance review

## Workflow Rule

Every workflow must define:

- Trigger
- User role
- Required data
- System action
- Notifications
- Documents created
- Financial impact
- Compliance impact
- Audit record
