# WORKFLOW FAILURE STATES 001

## Purpose

This document defines common failure states that workflows must handle.

## Common Failure States

### Applicant Workflow

- Incomplete application
- Failed screening
- Missing documents
- Applicant withdraws
- Unit becomes unavailable

### Maintenance Workflow

- Tenant submits unclear request
- Vendor does not respond
- Vendor cancels
- Parts unavailable
- Emergency escalates
- Tenant unavailable for access
- Repair incomplete

### Rent Collection Workflow

- Payment fails
- Partial payment received
- Late fee blocked by law
- Tenant disputes charge
- Payment processor delay

### Move-In Workflow

- Unit not ready
- Tenant does not appear
- Keys not available
- Inspection incomplete

### Move-Out Workflow

- Tenant abandons property
- Damage dispute
- Missing keys
- Deposit deadline approaching
- Unit requires major repair

## Failure State Rule

Every failure state must define:

- Who is notified
- What record is created
- What deadline applies
- What escalation path exists
- Whether compliance is affected
