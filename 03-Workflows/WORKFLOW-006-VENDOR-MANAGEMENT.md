# WORKFLOW 006: VENDOR MANAGEMENT

## Purpose

Defines how vendors are onboarded, assigned, evaluated, and paid.

## Trigger

A vendor is added or assigned to a work order.

## Main Actors

- Property Manager
- Vendor
- Owner
- System

## Workflow Steps

1. Vendor profile is created.
2. Vendor submits contact information.
3. Vendor uploads license, insurance, and tax documents.
4. System verifies required records.
5. Vendor status becomes Approved.
6. Vendor is assigned to Work Order.
7. Vendor completes service.
8. Vendor submits invoice.
9. Manager approves invoice.
10. Payment is issued.
11. Vendor performance is recorded.

## Records Created

- Vendor
- Person
- Role
- Document
- Work Order
- Invoice
- Financial Transaction
- Event
- Audit Log
