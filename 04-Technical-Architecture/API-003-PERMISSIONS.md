# API 003: PERMISSIONS

## Purpose

Defines authorization and access control.

## Permission Model

Access should be based on:

- Organization
- Role
- Property
- Unit
- Workflow
- Document sensitivity
- Financial visibility

## Example Permissions

- View property
- Edit property
- View lease
- Edit lease
- Approve payment
- Assign vendor
- View financial reports
- Send legal notice
- Manage users

## Rule

Permissions must be explicit, auditable, and role-based.
