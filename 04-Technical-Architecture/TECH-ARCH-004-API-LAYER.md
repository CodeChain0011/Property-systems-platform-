# TECH ARCH 004: API LAYER

## Purpose

Defines the API layer that connects frontend, backend, databases, integrations, and AI tools.

## API Responsibilities

- User authentication
- Property data access
- Workflow actions
- File upload coordination
- Payment coordination
- Notification triggering
- Reporting access
- AI assistant access
- External integrations

## API Design Rule

The API should be domain-based, not screen-based.

Examples:

- /properties
- /units
- /leases
- /work-orders
- /inspections
- /documents
- /financial-transactions
- /communications
