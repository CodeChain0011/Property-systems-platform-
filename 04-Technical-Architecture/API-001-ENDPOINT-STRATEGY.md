# API 001: ENDPOINT STRATEGY

## Purpose

Defines the first API design direction.

## API Style

Domain-based API structure.

## Example Domains

- /organizations
- /portfolios
- /properties
- /units
- /people
- /leases
- /applications
- /assets
- /work-orders
- /inspections
- /documents
- /financial-transactions
- /communications
- /events

## API Rule

APIs should follow business domains, not screen layouts.
